## 🚀 Как запустить:

### 1. Запустите WebSocket сервер (в PowerShell):

```powershell
$env:PATH += ";C:\Users\Qemetth\.bun\bin"
bun socket
```

Сервер будет работать на порту **3055** и доступен по адресу `ws://localhost:3055` или `ws://0.0.0.0:3055`

### 2. Установите плагин Figma:

1. Откройте Figma
2. Перейдите в **Plugins > Development > New Plugin**
3. Выберите **"Link existing plugin"**
4. Выберите файл `src/cursor_mcp_plugin/manifest.json` из этого проекта
5. Плагин появится в списке ваших development плагинов

### 3. Подключите плагин к серверу:

1. Запустите плагин в Figma
2. Используйте команду `join_channel` для подключения к каналу
3. WebSocket сервер должен быть запущен на порту 3055

### 4. Используйте в Cursor:

MCP сервер уже настроен в `.cursor/mcp.json`. Перезапустите Cursor, чтобы он подхватил конфигурацию.

## 📝 Важные замечания:

- WebSocket сервер должен быть запущен **до** подключения плагина Figma
- Порт настроен на **3055** как вы просили
- Hostname настроен на **0.0.0.0** для работы в Windows/WSL
- Bun добавлен в PATH для текущей сессии, для постоянного использования добавьте `C:\Users\Qemetth\.bun\bin` в системный PATH

## 🔧 Команды для работы:

- **Запуск WebSocket сервера**: `bun socket`
- **Сборка проекта**: `bun run build`
- **Установка зависимостей**: `bun install`
